# eventName

### Description

@short: Fires before... and so on

### Usage

~~~jsx {}
eventName: (
    id: string | number, 
    dir?: string
) => void;
~~~

### Parameters

The callback of the **beforeCollapse** event takes the following parameters:
- `id` - (required) the **ID** of the item that will be collapsed
- `dir` - (optional) the side the children will be hidden in relation to the parent shape ("left" or "right" for _type:"topic"_, otherwise - undefined)
- *other parameters with corresponding description*

### Returns

The callback returns ... (*if returns*)

:::info
For handling the inner events you can use the [Event Bus methods](path)
:::

### Example

~~~jsx {7-12}
// create JS Kanban
diagram.events.on("eventName",  function(id, dir){
    console.log("Collapsing "  + diagram.data.getItem(id).text, dir); 
    return  true;
});
~~~
